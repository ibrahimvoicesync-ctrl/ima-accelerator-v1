```sql
CREATE TABLE public.deals (
  id          uuid          PRIMARY KEY DEFAULT gen_random_uuid(),
  student_id  uuid          NOT NULL REFERENCES public.users(id) ON DELETE CASCADE,
  deal_number integer       NOT NULL,
  revenue     numeric(12,2) NOT NULL CHECK (revenue >= 0),
  profit      numeric(12,2) NOT NULL CHECK (profit >= 0),
  created_at  timestamptz   NOT NULL DEFAULT now(),
  updated_at  timestamptz   NOT NULL DEFAULT now(),
  CONSTRAINT deals_student_deal_number_key UNIQUE (student_id, deal_number)
);
```

**Section 2 -- deal_number trigger function** (per D-02, STATE.md D-01 locked):
```sql
CREATE OR REPLACE FUNCTION public.assign_deal_number()
RETURNS trigger
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public
AS $$
DECLARE
  v_next integer;
BEGIN
  SELECT COALESCE(MAX(deal_number), 0) + 1
    INTO v_next
    FROM public.deals
   WHERE student_id = NEW.student_id
     FOR UPDATE;

  NEW.deal_number := v_next;
  RETURN NEW;
END;
$$;
```

**Section 3 -- BEFORE INSERT trigger** (per D-02):
```sql
CREATE TRIGGER set_deal_number
  BEFORE INSERT ON public.deals
  FOR EACH ROW EXECUTE FUNCTION public.assign_deal_number();
```

**Section 4 -- updated_at trigger** (per D-16, reuse from 00001):

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Supabase client -> RLS | Untrusted queries from any authenticated user filtered by RLS policies |
| App API -> Postgres | API routes use admin client (bypasses RLS), so app-layer auth+role checks are primary defense; RLS is defense-in-depth |
| Concurrent inserts | Two simultaneous insert requests for the same student could race on deal_number |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-38-01 | Information Disclosure | deals RLS | mitigate | Student policies filter by `student_id = (select get_user_id())`; coach policies filter by `student_id IN (SELECT id FROM users WHERE coach_id = (select get_user_id()))` |
| T-38-02 | Tampering | deal_number | mitigate | BEFORE INSERT trigger with FOR UPDATE row lock prevents race-condition duplicate numbers; UNIQUE constraint is safety net |
| T-38-03 | Tampering | revenue/profit | mitigate | CHECK constraints `revenue >= 0`, `profit >= 0` prevent negative values at DB level; Phase 39 adds Zod validation at API level |
| T-38-04 | Elevation of Privilege | coach INSERT/UPDATE | mitigate | No coach_insert_deals or coach_update_deals policies exist; coaches cannot create or modify deals |
| T-38-05 | Tampering | deal_number caller override | accept | Trigger always overwrites deal_number on INSERT regardless of supplied value; Insert type marks it optional as signal to downstream Phase 39 to strip it from payload |
| T-38-06 | Denial of Service | RLS per-row function calls | mitigate | All policies use `(select get_user_role())` initplan pattern -- function evaluated once per query, not per row |
</threat_model>
