**Function 1: `get_owner_dashboard_stats()`**

No parameters. Returns jsonb with 4 keys:
```sql
CREATE OR REPLACE FUNCTION public.get_owner_dashboard_stats()
RETURNS jsonb
LANGUAGE plpgsql
STABLE
SECURITY DEFINER
SET search_path = public
AS $$
DECLARE
  v_today date := CURRENT_DATE;
  v_result jsonb;
BEGIN
  SELECT jsonb_build_object(
    'total_students',     (SELECT count(*) FROM users WHERE role = 'student' AND status = 'active'),
    'total_coaches',      (SELECT count(*) FROM users WHERE role = 'coach' AND status = 'active'),
    'active_today_count', (SELECT count(DISTINCT student_id) FROM work_sessions WHERE date = v_today),
    'reports_today',      (SELECT count(*) FROM daily_reports WHERE date = v_today AND submitted_at IS NOT NULL)
  ) INTO v_result;
  RETURN v_result;
END;
$$;
```

- `v_coach_rating numeric := 2.5` (matches `coachUnderperformingRating: 2.5`)
- `v_coach_window integer := 14` (matches `coachUnderperformingWindowDays: 14`)
- `v_report_inbox_days integer := 7` (matches `COACH_CONFIG.reportInboxDays: 7`)

Add SQL comments documenting the coupling: `-- SYNC: must match OWNER_CONFIG.alertThresholds in src/lib/config.ts`

For `p_role = 'owner'`:
1. Get all active students (`role='student', status='active'`) with `joined_at`
2. Get recent sessions within dropoff window (`date >= CURRENT_DATE - v_dropoff_days`)
3. Get recent reports within dropoff window (with `submitted_at IS NOT NULL`)
4. Build per-student last-active date (MAX of session date and report date)
5. Count inactive alerts (last active between inactive and dropoff cutoffs) — skip students whose account age in days < inactive threshold (grace period)
6. Count dropoff alerts (last active before dropoff cutoff OR no activity) — skip students whose account age < dropoff threshold (grace period)
7. Count unreviewed reports as 1 summary alert if any exist (`reviewed_by IS NULL AND submitted_at IS NOT NULL`)
8. Count coach underperformance: for each active coach, compute avg star_rating of their students' reports in the `v_coach_window` day window; if avg < `v_coach_rating`, add 1
9. Subtract dismissed alerts count from `alert_dismissals WHERE owner_id = p_user_id`
10. Return `jsonb_build_object('active_alerts', GREATEST(0, alert_count - dismissed_count))`

For `p_role = 'coach'`:
1. Count unreviewed reports from the coach's students within `v_report_inbox_days` days
2. Return `jsonb_build_object('unreviewed_reports', count)`

For any other role: return `'{}'::jsonb`
