  <action>
Create `src/lib/rate-limit.ts` with the following implementation:

1. Guard import at top: `import "server-only";` (same pattern as admin.ts — prevents accidental client-side import)

2. Import admin client: `import { createAdminClient } from "@/lib/supabase/admin";`

3. Export the result type:
   ```typescript
   export type RateLimitResult = {
     allowed: boolean;
     remaining: number;
     retryAfterSeconds: number;
   };
   ```

4. Export the async helper function with these exact parameters and defaults:
   ```typescript
   export async function checkRateLimit(
     userId: string,
     endpoint: string,
     maxRequests = 30,
     windowMinutes = 1
   ): Promise<RateLimitResult>
   ```

5. Function body:
   - Call `createAdminClient()` to get the admin Supabase client
   - Compute `windowStart` as `new Date(Date.now() - windowMinutes * 60 * 1000).toISOString()`
   - COUNT query: `admin.from("rate_limit_log").select("*", { count: "exact", head: true }).eq("user_id", userId).eq("endpoint", endpoint).gte("called_at", windowStart)`
   - Extract `count ?? 0` as `callCount`
   - If `callCount >= maxRequests`: return `{ allowed: false, remaining: 0, retryAfterSeconds: windowMinutes * 60 }`
   - Otherwise: INSERT `{ user_id: userId, endpoint }` into rate_limit_log, then return `{ allowed: true, remaining: maxRequests - callCount - 1, retryAfterSeconds: 0 }`

6. Error handling: Let errors propagate naturally. The calling route handler's try-catch (where present) will catch DB errors. Per CLAUDE.md hard rule, errors must not be silently swallowed — propagation is the correct behavior here. The rate limiter fails open (errors propagate, route handler's catch returns 500, which is acceptable — better than silently blocking legitimate users).

No Zod import needed — this module does not validate input.
