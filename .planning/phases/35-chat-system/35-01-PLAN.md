Create `src/lib/hooks/usePolling.ts`:
- "use client" directive at top (this is a React hook)
- Export `usePolling(callback: () => Promise<void> | void, intervalMs: number, enabled = true)`
- Use `useRef` to store the callback — update it on every render via a `useEffect` with `[callback]` dependency
- Second `useEffect` with `[intervalMs, enabled]` deps: if not enabled, return; call `callbackRef.current()` immediately on mount; then `setInterval(() => void callbackRef.current(), intervalMs)`; return cleanup `() => clearInterval(id)`
- CRITICAL: The callback ref pattern prevents stale closures (Pitfall 1 from research). Without it, the interval closure captures initial state.
- Import `useEffect, useRef` from "react"

Create `src/lib/chat-utils.ts`:
- Import `{ isToday, isYesterday, format, formatDistanceToNow } from "date-fns"`
- Export `formatDaySeparator(date: Date): string` — returns "Today" if isToday, "Yesterday" if isYesterday, else `format(date, "EEE, MMM d")` (e.g., "Mon, Mar 31")
- Export `formatMessageTime(date: Date): string` — returns `format(date, "h:mm a")` (e.g., "2:30 PM")
- Export `formatRelativeTime(date: Date): string` — returns `formatDistanceToNow(date, { addSuffix: true })` (e.g., "2 minutes ago")
- Export type `MessageGroup = { date: string; messages: MessageWithSender[] }` where `MessageWithSender` extends the messages Row type with `sender_name: string`
- Export type `ConsecutiveGroup = { senderId: string; messages: MessageWithSender[]; isCollapsed: boolean }`
- Export `groupMessagesByDay(messages: MessageWithSender[]): MessageGroup[]` — groups messages by calendar date (using `format(date, "yyyy-MM-dd")` as key), returns array sorted ascending by date
- Export `shouldShowTimestamp(current: MessageWithSender, previous: MessageWithSender | null): boolean` — returns true if no previous message or if gap >= 5 minutes (D-09)

- Export `MESSAGE_PAGE_SIZE = 30` (messages per page for cursor pagination)
- Export `MAX_MESSAGE_LENGTH = 2000` (CHAT-12)

Create `src/app/api/messages/route.ts` with two exports: GET and POST.

**GET handler** (polling endpoint — CHAT-03, CHAT-04, CHAT-08):
- NO CSRF check (GET endpoint)
- NO checkRateLimit() call (Pitfall 2: polling every 5s would hit 30 req/min cap)
- Auth: `const supabase = await createClient(); const { data: { user: authUser } } = await supabase.auth.getUser();` — return 401 if null
- Profile: `const admin = createAdminClient(); admin.from("users").select("id, role, coach_id").eq("auth_id", authUser.id).single()` — return 404 if null
- Role check: `if (!profile || !["coach", "student", "owner"].includes(profile.role))` return 403
- Accept query params via `request.nextUrl.searchParams`:
  - `student_id` (optional) — filter to specific conversation
  - `before` (optional) — cursor for pagination (ISO timestamp); if provided, add `.lt("created_at", before)`
  - `type` (optional, value "conversations") — when set, returns conversation summaries instead of messages
- Coach query: `admin.from("messages").select("*, sender:users!messages_sender_id_fkey(id, name)").eq("coach_id", profile.id).order("created_at", { ascending: false }).limit(MESSAGE_PAGE_SIZE)` — if student_id param present, add `.or(\`recipient_id.eq.${studentId},sender_id.eq.${studentId}\`)`
- Student query: `admin.from("messages").select("*, sender:users!messages_sender_id_fkey(id, name)").eq("coach_id", profile.coach_id)` then add `.or(\`and(recipient_id.eq.${profile.id},is_broadcast.eq.false),and(sender_id.eq.${profile.id},is_broadcast.eq.false),is_broadcast.eq.true\`)` — this gets all DMs to/from student AND all broadcasts from their coach
- Owner query: same as coach but accepts `coach_id` query param to filter
- Apply cursor pagination: if `before` param exists, add `.lt("created_at", before)`
- Always `.order("created_at", { ascending: false }).limit(30)`
- If `type=conversations` (coach only): fetch last 200 messages, then aggregate in JS — group by the non-coach participant (for DMs: the student), compute last message + unread count per student. Also fetch assigned students via `admin.from("users").select("id, name").eq("coach_id", profile.id).eq("status", "active")`. Return `{ conversations: ConversationSummary[] }`.
- Return `{ messages: data, hasMore: data.length === 30 }`
- Wrap in try/catch; catch logs `console.error("[GET /api/messages]", error)` and returns 500

**POST handler** (send message — CHAT-03, CHAT-04, CHAT-05, CHAT-12):
- CSRF: `const csrfError = verifyOrigin(request); if (csrfError) return csrfError;`
- Auth + profile lookup (same pattern as GET)
- Role check: coach or student only (owner excluded from sending)
- Rate limit: `const { allowed, retryAfterSeconds } = await checkRateLimit(profile.id, "/api/messages");` — return 429 if not allowed
- Body parse in try/catch: `body = await request.json()`
- Zod schema: `z.object({ content: z.string().min(1).max(2000), recipient_id: z.string().uuid().nullable(), is_broadcast: z.boolean().default(false) })`
- Validation: `const parsed = messageSchema.safeParse(body); if (!parsed.success)` return 400
- Business rules:
  - If coach: `coach_id = profile.id`, `sender_id = profile.id`
  - If coach + is_broadcast: `recipient_id` must be null; set `is_broadcast = true`
  - If student: `coach_id = profile.coach_id` (must not be null — return 400 "No coach assigned"), `sender_id = profile.id`, `recipient_id = profile.coach_id`, `is_broadcast` must be false (students cannot broadcast — return 403 if true)
- Insert: `admin.from("messages").insert({ coach_id, sender_id, recipient_id: parsed.data.recipient_id, is_broadcast: parsed.data.is_broadcast, content: parsed.data.content }).select().single()`
- Return 201 with `{ message: data }`
- Wrap in try/catch; catch logs and returns 500

Create `src/app/api/messages/read/route.ts` with PATCH export:

**PATCH handler** (mark as read — CHAT-07):
- CSRF: `const csrfError = verifyOrigin(request); if (csrfError) return csrfError;`
- Auth + profile lookup
- Role check: coach or student
- NO checkRateLimit() — fires at most once per conversation open, not user-controlled frequency
- Body parse + Zod: `z.object({ coach_id: z.string().uuid() })` — the conversation anchor
- Coach: update `read_at = new Date().toISOString()` WHERE `coach_id = parsed.data.coach_id AND recipient_id = profile.id AND read_at IS NULL` — marks messages sent TO the coach as read
- Student: update `read_at = new Date().toISOString()` WHERE `coach_id = parsed.data.coach_id AND (recipient_id = profile.id OR is_broadcast = true) AND read_at IS NULL` — marks DMs to student AND broadcasts as read
- Return `{ updated: count }` with 200
- Wrap in try/catch

Import `{ z } from "zod"` (NEVER from "zod/v4"). Import `{ NextRequest, NextResponse } from "next/server"`.
