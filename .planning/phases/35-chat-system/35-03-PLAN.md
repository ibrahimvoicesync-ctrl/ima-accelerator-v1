**`src/app/(dashboard)/coach/chat/page.tsx`:**
"use client" component (the entire page is client-side because of polling and state).

State:
- `conversations: ConversationSummary[]` — loaded from GET /api/messages?type=conversations
- `selectedStudentId: string | null` — which student conversation is open
- `isBroadcastMode: boolean` — whether broadcast composer is shown
- `messages: MessageWithSender[]` — messages for the selected conversation
- `hasMore: boolean` — whether older messages exist
- `isLoadingMore: boolean` — pagination in progress
- `showThread: boolean` — mobile view toggle (D-04)
- `isLoading: boolean` — initial load state

Refs:
- `bottomRef = useRef<HTMLDivElement>(null)` — scroll-to-bottom anchor
- `scrollContainerRef = useRef<HTMLDivElement>(null)` — for scroll position management
- `prevMessageCountRef = useRef(0)` — for auto-scroll detection (CHAT-09)

Polling:
- Use `usePolling` from `@/lib/hooks/usePolling` with `MESSAGE_POLL_INTERVAL` (5000ms)
- Conversation list polling: `fetchConversations()` — GET /api/messages?type=conversations
  - Check `response.ok` before parsing (Hard Rule 6)
  - Update `conversations` state
- Thread polling (when selectedStudentId is set): `fetchMessages()` — GET /api/messages?student_id={selectedStudentId}
  - On first load: set messages and scroll to bottom
  - On subsequent polls: if new messages detected (length increased), append to state and auto-scroll if user is near bottom
  - Auto-scroll guard (Pitfall 4): only scroll if `scrollContainer.scrollTop + scrollContainer.clientHeight >= scrollContainer.scrollHeight - 100`

Conversation selection:
- `onSelectStudent(studentId)`: set `selectedStudentId`, set `isBroadcastMode = false`, set `showThread = true` (mobile), clear messages, fetch thread messages, call PATCH /api/messages/read with `{ coach_id: currentUserId }` to mark as read (CHAT-07)
- `onSelectBroadcast()`: set `isBroadcastMode = true`, set `selectedStudentId = null`, set `showThread = true` (mobile)

Send message:
- `handleSend(content)`: POST /api/messages with `{ content, recipient_id: selectedStudentId, is_broadcast: false }`
  - Check response.ok before parsing
  - On success: append message to local state, scroll to bottom
- `handleBroadcast(content)`: POST /api/messages with `{ content, recipient_id: null, is_broadcast: true }`
  - On success: show sent confirmation, switch back to conversation list

Load older messages (CHAT-08):
- `loadOlderMessages()`:
  - Record `scrollHeight` before the API call
  - GET /api/messages?student_id={selectedStudentId}&before={messages[0].created_at}
  - Prepend older messages to state
  - After DOM update via `requestAnimationFrame`: `scrollContainer.scrollTop += (newScrollHeight - prevScrollHeight)` (Pitfall 3)

Layout (D-01):
- Desktop: two-panel split — left panel `w-[300px] border-r border-ima-border h-[calc(100vh-4rem)]` for ConversationList, right panel `flex-1 flex flex-col h-[calc(100vh-4rem)]` for MessageThread + ChatComposer
- Mobile (D-04): `showThread` state toggles between ConversationList (default) and thread view. Use `<div className="md:hidden">` and `<div className="hidden md:flex">` for responsive breakpoints.
- Mobile thread view: back button at top (ArrowLeft icon, min-h-[44px], `onClick={() => setShowThread(false)}`) with student name as header
- Mobile conversation list: full width when `!showThread`
- The page wrapper: `<div className="flex h-[calc(100vh-4rem)] -m-4 md:-m-8 bg-white">` (negative margin to fill the parent padding from layout.tsx)

Auth: The page must fetch the current user's profile to know `currentUserId` and role. Use a self-fetch approach: on mount, call a utility to get the user ID. Since this is a client component, fetch from an existing API or use Supabase client auth. Use `createClient()` from `@/lib/supabase/client` to call `supabase.auth.getUser()` on mount, then fetch profile from a lightweight endpoint or use the auth user ID directly.

IMPORTANT: Use `useRef` for toast and router references to keep them out of `useCallback` deps (stable ref pattern from codebase conventions). Import `useToast` and store as `toastRef`. Import `useRouter` and store as `routerRef`.

Error handling: every `fetch()` must check `response.ok`. Every catch block must call `toastRef.current.error(message)` or `console.error(...)` — never swallow errors (Hard Rule 5).
