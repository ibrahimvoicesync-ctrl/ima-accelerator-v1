## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding (`px-1.5`) |
| sm | 8px | Compact element spacing, tab gap (`gap-2`) |
| md | 16px | Default element spacing, page horizontal padding (`px-4`) |
| lg | 24px | Section padding (`p-6`, `space-y-6`) |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Empty state vertical padding (`py-16` → 64px — see exception) |
| 3xl | 64px | Not used on this page |

Exceptions:
- Alert card inner padding: 16px (`p-4`) — on-grid standard value.
- Touch targets: min-h 44px (`min-h-[44px]`) on all interactive elements per CLAUDE.md hard rule.
- Icon container: 40px × 40px (`w-10 h-10`) for milestone type icon — sub-44px non-interactive container only.
- Skeleton cards: height 44px per filter tab skeleton (`h-11 w-24`).

**Source:** CLAUDE.md hard rule 2 (44px touch targets); standard 8-point grid.

## Page Layout

Route: `/coach/alerts` (inside `(dashboard)` layout — Sidebar already rendered by layout).

```
┌──────────────────────────────────────────────────┐
│  [Bell icon] Milestone Alerts  [N active badge]  │  ← page heading row
│  "Your students' milestone achievements"          │  ← page subheading
├──────────────────────────────────────────────────┤
│  N active  |  N dismissed                         │  ← summary stats line
├──────────────────────────────────────────────────┤
│  [All]  [Active (N)]  [Dismissed]                 │  ← filter tabs
├──────────────────────────────────────────────────┤
│  ┌────────────────────────────────────────────┐  │
│  │ GROUP HEADER: "Student Name"  [Bulk Dismiss]│  │  ← per-student group header
│  │ ┌──────────────────────────────────────┐   │  │
│  │ │ [icon] Type badge  •  message text   │   │  │  ← alert row (Card bordered-left)
│  │ │         [View Student]  [Dismiss Alert]│  │  │
│  │ └──────────────────────────────────────┘   │  │
│  │ … more rows for same student …             │  │
│  └────────────────────────────────────────────┘  │
│  … next student group …                           │
└──────────────────────────────────────────────────┘
```

**Grouping contract:**
- Sort student groups: most recent `occurred_at` among any alert in the group descending (most active student first).
- Within a student group: sort alerts by `occurred_at` descending (newest first).
- Dismissed alerts are hidden when the "Active" tab is selected. All tabs still group by student.
- A student group header is **only shown** when the filtered view contains at least one alert for that student.

**Bulk Dismiss contract:**
- The "Dismiss All" button in a student group header only appears when at least one alert in the group is undismissed AND the current tab is "All" or "Active".
- Clicking it calls `POST /api/alerts/dismiss` sequentially for each undismissed alert key in the group (no bulk-dismiss API endpoint needed — reuse existing endpoint, fire in a `Promise.all`).
- Optimistic: entire student group visually transitions to dismissed state immediately; on any single failure, revert all and show single error toast.

**Sidebar badge:**
- The sidebar badge for Alerts renders `badgeCounts["alerts"]` from the layout's existing `get_sidebar_badges` RPC result.
- Capping logic: if `badgeCounts["alerts"] >= 10`, render the string `"9+"` instead of the raw number.
- The cap is applied in the Sidebar rendering logic (Sidebar.tsx), not in the RPC or page.
- Badge pill classes: `bg-ima-primary/10 text-ima-primary` (existing pattern from `Sidebar.tsx` line 263).

## States and Interactions

### Loading State

- Rendered by `src/app/(dashboard)/coach/alerts/loading.tsx` (already exists, matches design).
- 3 skeleton alert cards with icon placeholder, title line, body lines, and two action button skeletons.
- Filter tab skeletons: 3 × `h-11 w-24 rounded-lg`.
- All skeletons use `motion-safe:animate-pulse` (enforced by `Skeleton` component).

### Empty State (zero active alerts)

| Tab | Heading | Body |
|-----|---------|------|
| Active | "All caught up!" | "No active milestone alerts. Keep coaching!" |
| All | "No alerts yet" | "When your students hit milestones, they'll appear here." |
| Dismissed | "No dismissed alerts" | "Alerts you dismiss will be archived here for reference." |

- Use `EmptyState` component with `variant="default"` and `icon={<Bell className="h-6 w-6" />}`.
- Bell icon is `aria-hidden="true"` inside EmptyState.

### Alert Row — Active

- `Card variant="bordered-left"` with `className="border-l-ima-success"`.
- Unread dot: `w-2 h-2 rounded-full bg-ima-success` in top-right of card content area.
- `role="alert"` on the inner content div, `aria-label="milestone alert: {student name} — {type label}"`.
- Dismiss Alert `Button` rendered; loading state shows spinner via `loading={dismissingKey === key}`.
- Optimistic: on click, row immediately transitions to dismissed visual state (remove unread dot, switch to `variant="default"`, hide Dismiss Alert button). If API fails: revert + error toast.

### Alert Row — Dismissed

- `Card variant="default"` (no border-left, no colored left accent).
- No Dismiss Alert button.
- Greyed-out message text remains visible for context (`text-ima-text-secondary`).
- "View Student" link still active.

### Dismiss Error Toast

- Type: `"error"`, title: `"Failed to dismiss alert — please try again"`.
- Triggered when `POST /api/alerts/dismiss` returns non-OK or throws.
- Optimistic revert fires before toast.

### Bulk Dismiss Error Toast

- Type: `"error"`, title: `"Some alerts could not be dismissed — please try again"`.
- All successfully dismissed rows remain dismissed; only failed ones revert.

## Accessibility Contract

Every rule below is a CLAUDE.md hard rule — executor must verify each at implementation time.

| Requirement | Implementation |
|-------------|----------------|
| 44px touch targets | Every `<button>`, `<a>`, and `Link` with `min-h-[44px]` |
| aria-label on inputs | Not applicable (no text inputs on this page) |
| Alert role | `role="alert"` on each card's inner content div |
| Icon aria-hidden | `aria-hidden="true"` on all decorative Lucide icons |
| Filter tab aria-pressed | `aria-pressed={filter === key}` on each filter `<button>` |
| Empty state role | EmptyState renders `role="status"` (built into component) |
| Bulk dismiss focus | After bulk dismiss, focus moves to first remaining alert or empty state |
| Motion | All `animate-*` classes wrapped in `motion-safe:` |
| Loading skeleton | `aria-hidden="true"` on Skeleton divs (built into Skeleton component) |
| Page heading | Single `<h1>` per page ("Milestone Alerts") |
| Group heading | `<h2>` for each student group name |
| Student link | Descriptive text: "View {studentName}" set via `aria-label` on the link |
