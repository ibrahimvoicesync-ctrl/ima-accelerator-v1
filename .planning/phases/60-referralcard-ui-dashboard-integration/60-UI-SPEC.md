## Component Anatomy

### ReferralCard — State Machine

```
INITIAL  ──[click "Get My Link"]──►  LOADING  ──[fetch ok]──►  READY
                                        │
                                        └──[fetch error]──►  ERROR (re-enters INITIAL)
```

Primary focal point: "Get My Link" CTA button in INITIAL state; URL display row in READY state.

### State: INITIAL

Shell: `<div className="bg-ima-surface border border-ima-border rounded-xl p-6 mt-6">`

Contents:
1. **Heading** — `<h2 className="text-base font-semibold text-ima-text">`
   Text: `"Refer a Friend — Earn $500"`
2. **Description sentence** — `<p className="text-sm text-ima-text-secondary mt-1">`
   Text: `"Share your personal referral link with a friend and earn a $500 bonus when they join the IMA Accelerator program."`
   Rules: one sentence, professional, no emoji, no exclamation mark, no hype. Source: ROADMAP success criterion SC-1, REQUIREMENTS UI-02.
3. **Primary CTA button** — `<Button variant="primary" size="md" className="w-full mt-4" aria-label="Get My Link">`
   Label: `"Get My Link"`
   Min touch target: `h-11` (44px) — satisfied by Button size="md".

### State: LOADING

- Button becomes `<Button variant="primary" size="md" loading={true} disabled className="w-full mt-4">` — uses existing `Button` primitive's built-in `loading` prop which renders `<Spinner size="sm" />` + disables the button.
- Spinner renders `<Spinner size="sm" />` from `src/components/ui/Spinner.tsx` — already uses `motion-safe:animate-spin` internally. No custom spinner needed.
- Button label text hidden by loading state (Spinner takes visual precedence via the loading prop pattern).
- The existing Button component sets `aria-busy={loading}` automatically.
- Source: Spinner.tsx uses `motion-safe:animate-spin` (CLAUDE.md Hard Rule 1 satisfied).

### State: READY

Replaces the CTA button area with:

1. **URL display row** — `<div className="flex items-center gap-2 mt-4 bg-ima-surface-light rounded-lg px-2 py-2">`
   - `<span className="flex-1 text-sm text-ima-text truncate font-mono">{shortUrl}</span>`
   - **Copy button** (icon-only, toggles):
     - Note: Copy resting state is icon-only by design (aria-label fallback); visible "Copied!" text appears only on successful copy. Share is permanently icon-only (aria-label fallback) — hidden entirely when `navigator.share` is unavailable.
     - Default: `<button className="shrink-0 min-h-[44px] min-w-[44px] flex items-center justify-center rounded-lg hover:bg-ima-surface-accent motion-safe:transition-colors" aria-label="Copy referral link">`
       - Icon: `<Copy className="h-4 w-4 text-ima-text-secondary" aria-hidden="true" />`
     - Copied (2-second toggle): label becomes `"Copied to clipboard"`, icon swaps to `<Check className="h-4 w-4 text-ima-success" aria-hidden="true" />`, button text `"Copied!"` appears beside icon (`text-xs text-ima-success font-semibold`)
     - After 2 seconds: reverts to default Copy state via `setTimeout` + `useRef` cleanup.
   - **Share button** (icon-only, conditionally rendered):
     - Condition: rendered only when `navigator.share` is available — detected via `useEffect` + boolean state (`shareSupported`), never read during SSR render. Defaults to `false`.
     - `<button className="shrink-0 min-h-[44px] min-w-[44px] flex items-center justify-center rounded-lg hover:bg-ima-surface-accent motion-safe:transition-colors" aria-label="Share referral link">`
     - Icon: `<Share2 className="h-4 w-4 text-ima-text-secondary" aria-hidden="true" />`
     - On click: calls `navigator.share({ url: shortUrl, title: "IMA Accelerator Referral" })` wrapped in try/catch (abort errors silently ignored; other errors toasted).

All icons in READY state: `aria-hidden="true"` (decorative).
All interactive elements: `min-h-[44px] min-w-[44px]` (CLAUDE.md Hard Rule 2).

### Error Handling

- Fetch error (network error or `!response.ok`): call `useToast().toast({ type: "error", title: "Could not generate your link", description: "Please try again." })` AND `console.error("[ReferralCard]", error)`.
- Component returns to INITIAL state — spinner clears, button re-enables.
- `response.ok` is checked before any `await response.json()` call (CLAUDE.md Hard Rule 6).
- Never silently swallowed (CLAUDE.md Hard Rule 5).

### Inline Error (alternative — NOT used)

Toast variant is preferred (matches existing DealsClient, WorkTrackerClient, and ReportForm error patterns). Inline error is not used.

## CLAUDE.md Hard Rule Compliance Checklist

| Rule | Requirement | How Satisfied |
|------|-------------|---------------|
| 1 — motion-safe | Every `animate-*` uses `motion-safe:` | `Spinner` uses `motion-safe:animate-spin` internally; Button hover uses `motion-safe:transition-colors`; no raw `animate-spin` in new code |
| 2 — 44px touch targets | Every interactive element needs `min-h-[44px]` | "Get My Link": `Button size="md"` = `h-11`; Copy button: `min-h-[44px] min-w-[44px]`; Share button: `min-h-[44px] min-w-[44px]` |
| 3 — Accessible labels | Every input needs aria-label or label+htmlFor | No inputs. Icon-only buttons: `aria-label` on Copy and Share. |
| 4 — Admin client in API routes | Not applicable to UI component | n/a (component calls API route, not DB directly) |
| 5 — Never swallow errors | Every catch must toast or console.error | Fetch error: toast + console.error; navigator.share abort: silently ignored (user-initiated cancel is not an error); other share errors: console.error |
| 6 — Check response.ok | Every fetch checks response.ok | `if (!response.ok)` gate before `response.json()` |
| 7 — Zod import | `import { z } from "zod"` | Not used in UI component (no request body to validate); API route already handles Zod |
| 8 — ima-* tokens only | All colors use ima-* tokens, never hardcoded hex/gray | All classes use `ima-*` tokens. `text-white` only on primary button (bg-ima-primary background). |
