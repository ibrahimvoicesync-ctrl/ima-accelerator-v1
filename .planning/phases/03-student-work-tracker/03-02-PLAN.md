**1. Create `src/components/student/WorkTimer.tsx`:**

A "use client" component that renders the hero circular countdown timer.

Props interface:
```typescript
interface WorkTimerProps {
  sessionId: string;
  startedAt: string;       // ISO timestamp from DB
  cycleNumber: number;     // 1-4
  totalSeconds: number;    // WORK_TRACKER.sessionMinutes * 60 = 2700
  onComplete: () => void;  // called when timer hits 0
}
```

Implementation details:
- SVG circular progress ring: `size = 280`, `strokeWidth = 8`, `radius = (size - strokeWidth) / 2`, `circumference = 2 * Math.PI * radius`
- `strokeDashoffset = circumference * (1 - progress)` where `progress = remainingSeconds / totalSeconds`
- Ring SVG has `-rotate-90` class, track ring uses `text-ima-border`, progress ring uses `text-ima-primary`
- Progress ring transition: `className="text-ima-primary motion-safe:transition-[stroke-dashoffset] duration-1000 ease-linear"`
- SVG has `aria-hidden="true"` (decorative — the timer text is the accessible content)
- Timer text overlay centered in SVG: large `text-4xl md:text-5xl font-mono font-bold text-ima-text` showing MM:SS
- Below timer text: `text-sm text-ima-text-secondary` showing "Cycle {cycleNumber} of 4"
- Outer container: `role="timer"` with `aria-label="{minutes} minutes and {seconds} seconds remaining in Cycle {cycleNumber} of 4"`
- Screen reader live region: `<div aria-live="polite" className="sr-only">` announces every 15 seconds
- `calcRemaining` function using `useCallback`:
  ```
  const elapsed = Math.floor((Date.now() - new Date(startedAt).getTime()) / 1000);
  return Math.max(0, totalSeconds - elapsed);
  ```
- On mount: `setRemainingSeconds(calcRemaining())` to restore from started_at
- `useEffect` with `setInterval(1000)`: tick every second, call `calcRemaining()`, if <= 0 clear interval and call `onCompleteRef.current()`
- Store `onComplete` in a ref (`onCompleteRef`) to avoid tearing down the interval on callback identity change
- Cleanup: `return () => clearInterval(interval)` in the useEffect
- Browser tab title: `document.title = \`${timeStr} - Work Tracker | IMA\`` during tick. Reset to `"Work Tracker | IMA"` in useEffect cleanup.

**1. Create `src/components/student/WorkTrackerClient.tsx`:**

A "use client" component that orchestrates the entire work tracker page. This is the main interactive island.

Props:
```typescript
import type { Database } from "@/lib/types";
type WorkSession = Database["public"]["Tables"]["work_sessions"]["Row"];

interface WorkTrackerClientProps {
  initialSessions: WorkSession[];
}
```

State management:
- `sessions` state initialized from `initialSessions` prop
- Derived values computed from sessions:
  - `completedCount = sessions.filter(s => s.status === "completed").length`
  - `activeSession = sessions.find(s => s.status === "in_progress")`
  - `pausedSession = sessions.find(s => s.status === "paused")`
  - `allComplete = completedCount >= WORK_TRACKER.cyclesPerDay`
  - `nextCycleNumber = completedCount + 1` (capped at cyclesPerDay for display)
  - `totalMinutesWorked = sessions.filter(s => s.status === "completed").reduce((sum, s) => sum + s.duration_minutes, 0)`
- `isLoading` boolean for button disabled states during mutations
- `showAbandonConfirm` boolean for inline abandon confirmation
- Use `useRef` for `router` (from `next/navigation` `useRouter()`) to keep stable reference in callbacks

Stale session auto-abandon on mount:
- In a `useEffect` with empty deps, check `initialSessions` for any session with `status === "in_progress"` or `status === "paused"` where `date < getToday()`
- For each stale session, fire a silent PATCH to `/api/work-sessions/${session.id}` with `{ status: "abandoned" }`
- After all stale abandons complete, call `router.refresh()` to re-fetch from server

Mutation handlers (all async, set `isLoading` true at start, false at end):

**handleStart():**
- POST to `/api/work-sessions` with body `{ date: getToday(), cycle_number: nextCycleNumber }`
- Check `response.ok` before parsing JSON (CLAUDE.md hard rule)
- On success: `router.refresh()` — server component re-fetches and passes new initialSessions
- On error: show toast (use a simple `alert()` or inline error state — no toast library in stack yet; if sonner is installed use it, otherwise use window alert or a useState error message)

**handleComplete(sessionId: string):**
- PATCH to `/api/work-sessions/${sessionId}` with `{ status: "completed", duration_minutes: WORK_TRACKER.sessionMinutes }`
- Check response.ok
- On success: `router.refresh()`, show success feedback
- On error: If 400 with "Cannot transition" — silently ignore (race condition with auto-complete)

**handlePause(sessionId: string):**
- PATCH to `/api/work-sessions/${sessionId}` with `{ status: "paused" }`
- Check response.ok
- On success: `router.refresh()`

**handleResume(sessionId: string):**
- PATCH to `/api/work-sessions/${sessionId}` with `{ status: "in_progress" }`
- Check response.ok
- On success: `router.refresh()`

**handleAbandon(sessionId: string):**
- First check grace period: if `Date.now() - new Date(activeSession.started_at).getTime() < WORK_TRACKER.abandonGraceSeconds * 1000`:
  - Show inline confirmation (`showAbandonConfirm = true`)
  - Return (don't send PATCH yet)
- If past grace period or `showAbandonConfirm` is already true (user confirmed):
  - PATCH to `/api/work-sessions/${sessionId}` with `{ status: "abandoned" }`
  - Check response.ok
  - On success: `router.refresh()`, reset `showAbandonConfirm = false`

Render logic (top to bottom):

**If `allComplete`:**
- Green celebration card: `bg-green-50 border-green-200 rounded-xl p-6 text-center`
- Heading: `text-xl font-bold text-green-800` "All 4 cycles complete!"
- Subtext: `text-green-700` showing `formatHours(totalMinutesWorked)` worked today
- Nudge: `text-sm text-green-600 mt-2` "Great work! Don't forget to submit your daily report."
- CTA: Link to `/student/report` styled as primary button: `bg-ima-primary text-white rounded-lg px-6 min-h-[44px] font-medium hover:bg-ima-primary-hover`

**If `activeSession` (timer running):**
- `<WorkTimer sessionId={activeSession.id} startedAt={activeSession.started_at} cycleNumber={activeSession.cycle_number} totalSeconds={WORK_TRACKER.sessionMinutes * 60} onComplete={() => handleComplete(activeSession.id)} />`
- Action buttons below timer in a flex row with gap-3, centered:
  - Pause button: `bg-amber-100 text-amber-700 hover:bg-amber-200 rounded-lg px-6 min-h-[44px] font-medium` — onClick calls `handlePause(activeSession.id)`
  - Complete button: `bg-green-600 text-white hover:bg-green-700 rounded-lg px-6 min-h-[44px] font-medium` — onClick calls `handleComplete(activeSession.id)`, disabled when isLoading
  - Abandon button: `text-red-600 hover:bg-red-50 rounded-lg px-4 min-h-[44px] text-sm` — onClick calls `handleAbandon(activeSession.id)`
- If `showAbandonConfirm`: render inline confirmation below buttons: `bg-red-50 rounded-lg p-3 text-sm text-red-700` with "Are you sure? Less than 5 minutes have passed." and a "Confirm Abandon" button

**If `pausedSession` (timer paused, no active):**
- Show paused state: large text showing remaining time from `formatPausedRemaining(pausedSession.started_at, pausedSession.paused_at!, WORK_TRACKER.sessionMinutes)`
- Label: "Cycle {pausedSession.cycle_number} paused"
- Resume button (prominent): `bg-ima-primary text-white rounded-lg px-6 min-h-[44px] font-medium hover:bg-ima-primary-hover` — calls `handleResume(pausedSession.id)`
- Abandon button: same styling as in active state

**Else (idle — no active or paused session):**
- If `completedCount < WORK_TRACKER.cyclesPerDay`:
  - Start button: `bg-ima-primary text-white rounded-xl px-8 min-h-[56px] text-lg font-semibold hover:bg-ima-primary-hover` — calls `handleStart()`, disabled when isLoading
  - Label: `text-lg font-semibold text-ima-text` `"Start Cycle {nextCycleNumber}"`
  - Subtitle: `text-sm text-ima-text-secondary` `"{completedCount} of 4 cycles done"`

**Cycle progress grid (always rendered below the main action area):**
- Container: `grid grid-cols-1 sm:grid-cols-2 gap-3 mt-6`
- Render 4 CycleCards (1 through WORK_TRACKER.cyclesPerDay):
  - Find session for this cycle_number in `sessions`
  - If found: render CycleCard with actual status and time info
    - completed: timeInfo = "45 min"
    - in_progress: timeInfo = "{remaining} left" (compute from started_at)
    - paused: timeInfo = formatPausedRemaining(...) + " left", onResume = () => handleResume(session.id)
    - abandoned: timeInfo = "Abandoned"
  - If not found: render CycleCard with status="pending", timeInfo="Pending"

**2. Create `src/app/(dashboard)/student/work/page.tsx`:**

Server component that fetches today's sessions and passes to client island.

```typescript
import { requireRole } from "@/lib/session";
import { createAdminClient } from "@/lib/supabase/admin";
import { WorkTrackerClient } from "@/components/student/WorkTrackerClient";
import type { Database } from "@/lib/types";

type WorkSession = Database["public"]["Tables"]["work_sessions"]["Row"];

export default async function WorkPage() {
  const user = await requireRole("student");
  const admin = createAdminClient();
  const today = new Date().toISOString().split("T")[0];

  const { data: sessions, error } = await admin
    .from("work_sessions")
    .select("*")
    .eq("student_id", user.id)
    .eq("date", today)
    .order("cycle_number", { ascending: true });

  if (error) {
    console.error("[work page] Failed to load sessions:", error);
  }

  return (
    <div className="max-w-2xl mx-auto px-4">
      <h1 className="text-2xl font-bold text-ima-text mb-1">Work Tracker</h1>
      <p className="text-sm text-ima-text-secondary mb-6">
        Track your daily 45-minute work cycles
      </p>
      <WorkTrackerClient initialSessions={(sessions ?? []) as WorkSession[]} />
    </div>
  );
}
```

Key details:
- Uses `requireRole("student")` (not getSessionUser — includes role enforcement)
- Uses `createAdminClient()` for the query (admin client in server code per CLAUDE.md)
- Filters by `student_id = user.id` (defense-in-depth)
- Passes `sessions ?? []` to handle null data gracefully
- `max-w-2xl mx-auto` centers content for the timer hero layout
- px-4 for mobile padding per CLAUDE.md
