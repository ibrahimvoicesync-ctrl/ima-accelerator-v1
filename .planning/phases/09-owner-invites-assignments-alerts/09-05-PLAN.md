**Step 2 — Add grace period check in the student classification loop (lines 81-117):**
Before the current `if (!last || last < dropoffCutoff)` block, add a grace period check. The logic change:

```typescript
for (const student of students) {
  const last = lastActive[student.id];

  // Grace period: skip alerts for students whose account is younger than the threshold
  const accountAgeMs = nowMs - new Date(student.joined_at).getTime();
  const accountAgeDays = accountAgeMs / 86400000;

  if (!last || last < dropoffCutoff) {
    // Dropoff: 7+ days or never active — BUT skip if account < 7 days old
    if (accountAgeDays < thresholds.studentDropoffDays) continue;

    const key = `student_dropoff:${student.id}:${isoWeek}`;
    // ... rest of dropoff alert creation (unchanged)
  } else if (last < inactiveCutoff) {
    // Inactive: 3-6 days — BUT skip if account < 3 days old
    if (accountAgeDays < thresholds.studentInactiveDays) continue;

    const key = `student_inactive:${student.id}:${today}`;
    // ... rest of inactive alert creation (unchanged)
  }
}
```

The key insight: if a student's account is younger than `studentDropoffDays` (7 days), they CANNOT be a dropoff. If younger than `studentInactiveDays` (3 days), they CANNOT be inactive. This naturally handles the "never logged any activity" case because new accounts get a grace period before being flagged.

Do NOT change any other alert logic (unreviewed reports, coach underperformance). Those are unaffected by this bug.

  if (!last || last < dropoffCutoff) {
    if (accountAgeDays < thresholds.studentDropoffDays) continue;

    if (accountAgeDays < thresholds.studentInactiveDays) continue;
    alertCount++; // inactive

}
