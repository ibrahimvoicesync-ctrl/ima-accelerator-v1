**Step 2: Add month bounds calculation**

After the `const today = getTodayUTC();` line, add:

```typescript
// Month-scoped calendar data: validate ?month=YYYY-MM or default to current month
const monthStr = typeof month === "string" && /^\d{4}-\d{2}$/.test(month) ? month : today.slice(0, 7);
const firstDay = `${monthStr}-01`;
const lastDayDate = new Date(firstDay + "T00:00:00Z");
```

**Step 3: Replace old sessions/reports queries with month-scoped versions**

In the `Promise.all` array, REPLACE the existing sessions query:
```typescript
// OLD:
admin.from("work_sessions")
  .select("id, date, cycle_number, status, duration_minutes")
  .eq("student_id", student.id)
  .order("date", { ascending: false })
  .limit(120),
```
with:
```typescript
// NEW — month-scoped calendar sessions
admin.from("work_sessions")
  .select("id, date, cycle_number, status, duration_minutes, session_minutes")
  .eq("student_id", student.id)
  .gte("date", firstDay)
  .lte("date", lastDay)
  .order("date")
  .order("cycle_number"),
```

Note the additions: `session_minutes` in select, `.gte("date", firstDay).lte("date", lastDay)` for month scoping, removed `.limit(120)`, added `.order("cycle_number")` secondary sort, changed to ascending order.

REPLACE the existing reports query:

**Step 6: Update recent ratings for at-risk**

The at-risk `recentRatings` computation currently uses `reports` (the old full list). It needs the last 7 days of reports for star ratings. Since `calendarReports` is month-scoped, we can filter it:

```typescript
  .map((r) => r.star_rating!);
```

HOWEVER, if the current month view is not the current month (user navigated to a past month), `calendarReports` won't have the last 7 days. To handle this correctly, add a dedicated at-risk reports query ONLY IF needed. The simplest approach: use the `latestReportResult` data range. Actually, the cleanest solution is to add a small query for recent ratings:

```typescript
// At-risk: recent reports for star rating computation (last 7 days)

  .select("date, star_rating")

  .not("star_rating", "is", null),
```

BUT `sevenDaysAgo` is computed after Promise.all currently. Move the `sevenDaysAgo` computation BEFORE Promise.all:

```typescript
const nowMs = Date.now();
```

Then add the recent ratings query to Promise.all and destructure as `recentRatingsResult`. Use it for at-risk:
