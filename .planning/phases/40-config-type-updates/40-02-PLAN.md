
Key details:
- `revenue` and `profit` are `string | number` (not just `number`) because PostgreSQL `numeric(12,2)` comes back as string from PostgREST. This matches v1.5 decision D-02 in STATE.md: "numeric(12,2) revenue/profit declared as string | number in types.ts -- coerce with Number() at every arithmetic site."
- `deal_number` is `number` (integer column).
- `deal_number` is optional in Insert (`deal_number?: number`) because the BEFORE INSERT trigger assigns it automatically.
- `deal_number` is optional in Update (`deal_number?: number`) because students never update it.
- Single Relationship: `deals_student_id_fkey` referencing `users.id`.

Do NOT modify any other part of types.ts. Only add the deals block.
