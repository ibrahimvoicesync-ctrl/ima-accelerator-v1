Create `src/app/api/auth/signout/route.ts` as a POST route handler.

**Full implementation:**
```typescript
import { createClient } from "@/lib/supabase/server";
import { NextResponse } from "next/server";

export async function POST(request: Request) {
  const supabase = await createClient();
  const { error } = await supabase.auth.signOut();

  if (error) {
    console.error("[signout] Failed to sign out:", error);
  }

  const { origin } = new URL(request.url);
  return NextResponse.redirect(`${origin}/login`, { status: 302 });
}
```

**Key points:**
- POST method (not GET) to prevent CSRF via link/img tags
- Uses the SSR client (not admin) for signOut -- this clears the session cookies
- Always redirects to /login regardless of error (user wants to leave)
- Logs error but doesn't block sign-out (graceful degradation)
- Uses request.url origin to build absolute redirect URL
- Sign-out redirect goes straight to /login (Claude's discretion per CONTEXT.md)

**Note on proxy.ts config matcher:** The proxy.ts config excludes API routes (`api/` prefix), so this route is not intercepted by the proxy. This is correct -- API routes handle their own auth.
