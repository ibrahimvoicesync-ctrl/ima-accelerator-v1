    ```typescript
    const addResourceSchema = z.object({
      title: z.string().min(1).max(255),
      url: z.string().url().max(2048),
      comment: z.string().max(1000).optional(),
      is_pinned: z.boolean().optional(),
    });

    const deleteResourceSchema = z.object({
      id: z.string().uuid(),
    });
    ```

    **GET handler (no CSRF, no rate limit — read endpoint per anti-pattern rule from RESEARCH.md):**
    1. Auth check: `createClient()` -> `getUser()` -> 401 if no user
    2. Profile lookup: `createAdminClient()` -> select id, role from users where auth_id
    3. Role check: allow `["owner", "coach", "student"]` only, 403 otherwise
    4. Query: `admin.from("resources").select("*, created_by_user:users!resources_created_by_fkey(name)").order("is_pinned", { ascending: false }).order("created_at", { ascending: false })`
    5. Return `{ resources: data ?? [] }`
    6. Wrap entire handler in try/catch with `console.error("[GET /api/resources]", err)` and 500 response

    **POST handler (CSRF + rate limit — mutation):**
    1. CSRF: `const originOk = await verifyOrigin(request); if (!originOk) return 403`
    2. Auth check (same pattern as GET)
    3. Profile lookup
    4. Role check: allow `["owner", "coach"]` only, 403 otherwise (students are read-only per RES-05)
    5. Rate limit: `const { allowed } = await checkRateLimit(profile.id); if (!allowed) return 429`
    6. Body parse: `try { body = await request.json() } catch { return 400 "Invalid JSON" }`
    7. Zod validate: `const parsed = addResourceSchema.safeParse(body); if (!parsed.success) return 400 with parsed.error.issues`
    8. Insert: `admin.from("resources").insert({ title: parsed.data.title, url: parsed.data.url, comment: parsed.data.comment ?? null, is_pinned: parsed.data.is_pinned ?? false, created_by: profile.id }).select("*, created_by_user:users!resources_created_by_fkey(name)").single()`
    9. Return `{ resource: data }` with status 201
    10. Wrap in try/catch with `console.error("[POST /api/resources]", err)` and 500 response

    **DELETE handler (CSRF + rate limit — mutation):**
    1. CSRF: verifyOrigin -> 403
    2. Auth check
    3. Profile lookup
    4. Role check: allow `["owner", "coach"]` only
    5. Rate limit
    6. Body parse with try/catch
    7. Zod validate with deleteResourceSchema
    8. **Ownership check for coach (per Pitfall 6):**
       ```typescript
       if (profile.role === "coach") {
         const { data: existing } = await admin
           .from("resources")
           .select("created_by")
           .eq("id", parsed.data.id)
           .single();
         if (!existing || existing.created_by !== profile.id) {
           return NextResponse.json({ error: "Forbidden" }, { status: 403 });
         }
       }
       ```
    9. Delete: `admin.from("resources").delete().eq("id", parsed.data.id)`
    10. Return `{ success: true }` with status 200
    11. Wrap in try/catch with `console.error("[DELETE /api/resources]", err)` and 500 response
