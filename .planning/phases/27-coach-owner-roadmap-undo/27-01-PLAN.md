**Zod schema:**
```typescript
const undoSchema = z.object({
  studentId: z.string().uuid(),
  step_number: z.number().int().min(1).max(ROADMAP_STEPS.length),
});
```

**Auth chain (in this exact order):**
1. `verifyOrigin(request)` -- return its response if non-null
2. `createClient()` -> `supabase.auth.getUser()` -- 401 if no authUser
3. `createAdminClient()` -> `admin.from("users").select("id, role").eq("auth_id", authUser.id).single()` -- 404 if no profile
4. Role guard: `if (!profile || (profile.role !== "coach" && profile.role !== "owner"))` -> 403 "Forbidden"
5. `checkRateLimit(profile.id, "/api/roadmap/undo")` -- 429 with Retry-After header if !allowed
6. `request.json()` in try/catch -- 400 "Invalid JSON body" on catch
7. `undoSchema.safeParse(body)` -- 400 with first issue message on failure
8. Destructure `{ studentId, step_number }` from `parsed.data`

**Ownership check (per D-03 response shape, UNDO-01, UNDO-02):**
- Query `admin.from("users").select("id, coach_id").eq("id", studentId).eq("role", "student").single()`
- If error or no student -> 404 "Student not found"
- If `profile.role === "coach" && student.coach_id !== profile.id` -> 403 "Forbidden"
- Owner falls through (no check needed)

**Revert step N (UNDO-04 cascade safety):**
```typescript
const { data: reverted, error: revertError } = await admin
  .from("roadmap_progress")
  .update({ status: "active", completed_at: null })
  .eq("student_id", studentId)
  .eq("step_number", step_number)
  .eq("status", "completed")
  .select()
  .single();
```
If `revertError || !reverted` -> 400 "Step not found or already reverted"

**Cascade re-lock N+1 (only if N+1 is active, per UNDO-04):**
```typescript
let relocked = null;
if (step_number < ROADMAP_STEPS.length) {
  const { data: nextStep } = await admin
    .from("roadmap_progress")
    .update({ status: "locked" })
    .eq("student_id", studentId)
    .eq("step_number", step_number + 1)
    .eq("status", "active")
    .select()
    .single();
  relocked = nextStep ?? null;
}
```
The `.eq("status", "active")` guard prevents accidentally re-locking a completed N+1 step.

**Audit log INSERT:**
```typescript
await admin.from("roadmap_undo_log").insert({
  actor_id: profile.id,
  actor_role: profile.role,
  student_id: studentId,
  step_number,
});
```

**Response (per D-03):**

return NextResponse.json({

});
