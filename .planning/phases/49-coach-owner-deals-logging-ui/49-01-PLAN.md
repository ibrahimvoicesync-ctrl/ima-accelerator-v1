# Plan 49-01 — Shared `formatDealLoggedBy` helper + `DealAttributionChip` + consumer wiring

## Objective

Produce the single source of truth for deal attribution so every deals table across the app renders a consistent chip whose label is gated by one pure helper. Apply that chip everywhere deals are listed today (student self list, coach student-detail Deals tab, owner student-detail Deals tab, student analytics Deal History). This plan adds NO new buttons — plan 49-02 handles the Add Deal flow. This plan also adds the `logged_by` user name map to the two student-detail pages so tabs can render attribution.

## Helper contract (`src/lib/deals-attribution.ts`)

```ts
import type { Database } from "@/lib/types";

type Deal = Pick<
  Database["public"]["Tables"]["deals"]["Row"],
  "logged_by" | "student_id"
>;

export type ViewerRole = "student" | "student_diy" | "coach" | "owner";

export interface LoggedByUser {
  id: string;
  name: string;                               // full name from users.name
  role: "student" | "student_diy" | "coach" | "owner";
}

export type AttributionVariant = "self" | "coach" | "owner" | "unknown";

export interface AttributionResult {
  label: string;                              // "You" | "Michael" | "Owner: Abu Lahya" | "Unknown"
  variant: AttributionVariant;
  ariaLabel: string;                          // "Logged by {fullName}" or "Logged by you"
}

export function formatDealLoggedBy(
  deal: Deal,
  viewerRole: ViewerRole,
  viewerId: string,
  userMap: Record<string, LoggedByUser>,
): AttributionResult;
```

### Behavior rules

1. If `deal.logged_by === viewerId` → `{ label: "You", variant: "self", ariaLabel: "Logged by you" }`.
2. Else look up `user = userMap[deal.logged_by]`.
   - If `user` missing → `{ label: "Unknown", variant: "unknown", ariaLabel: "Logged by unknown user" }` and `console.warn` ONCE per render tree (use a module-level `WeakSet` keyed on userMap — actually, just call `console.warn("[deals-attribution] missing user for logged_by", deal.logged_by)` — never swallow).
   - If `user.role === "coach"` → `{ label: firstName(user.name), variant: "coach", ariaLabel: "Logged by " + user.name }`.
   - If `user.role === "owner"` → `{ label: "Owner: " + firstName(user.name), variant: "owner", ariaLabel: "Logged by " + user.name }`.
   - If `user.role === "student" | "student_diy"` → `{ label: firstName(user.name), variant: "unknown", ariaLabel: "Logged by " + user.name }` (student logged by another student shouldn't happen post-Phase 45 but render defensively).
3. `firstName(name)` returns the first whitespace-separated token, falling back to the full string if empty.

## Component contract (`src/components/shared/DealAttributionChip.tsx`)

- `"use client"` NOT required — it's purely presentational and may be a Server Component.
- Props: `{ deal, viewerRole, viewerId, userMap }`.
- Renders a `<span role="status" aria-label={result.ariaLabel}>` with visual variants:
  - `self` → `bg-ima-surface-accent text-ima-primary`
  - `coach` → `bg-ima-surface-light text-ima-text-secondary`
  - `owner` → `bg-ima-surface-light text-ima-text-secondary`
  - `unknown` → `bg-ima-surface-light text-ima-text-muted`
- Shared classes: `inline-flex items-center rounded-full px-2.5 py-1 text-xs font-medium`.
- No motion, no interactive affordances — strictly informational.

## Page-level userMap fetch

In the two student-detail server pages, after `dealsData` is fetched:

```ts
const loggedByIds = Array.from(new Set(dealsData.map((d) => d.logged_by).filter(Boolean)));
const { data: loggedByUsers } = loggedByIds.length > 0
  ? await admin.from("users").select("id, name, role").in("id", loggedByIds)
  : { data: [] };
const userMap: Record<string, LoggedByUser> = {};
for (const u of loggedByUsers ?? []) {
  userMap[u.id] = { id: u.id, name: u.name, role: u.role };
}
```

Pass `userMap` and `viewerId` (from `user.id`) into the client components.

For `src/app/(dashboard)/owner/students/[studentId]/page.tsx` — owner route only has `await requireRole("owner")` (no id returned). Update the call to `const owner = await requireRole("owner");` then pass `owner.id` as `viewerId`.

## Analytics page chip migration

In `src/app/(dashboard)/student/analytics/AnalyticsClient.tsx`:
- Delete the inline `AttributionChip` function.
- Replace call with `<DealAttributionChip deal={{ logged_by: d.logged_by, student_id: "" }} viewerRole="student" viewerId={viewerId} userMap={userMap} />`.
- The analytics RPC already returns `logged_by`, `logger_role`, and `is_self`. Build `userMap` and `viewerId` from the page server component by:
  - Passing `viewerId` as a new prop from `page.tsx`.
  - Deriving a `userMap` on the server from `data.deals` `logged_by` ids (same pattern as student-detail pages).
- If upgrading the analytics surface to the shared chip introduces type friction (RPC row vs helper's `Deal` shape), widen the helper input to `Pick<..., "logged_by">` only — `student_id` is not read by the helper and can be dropped from the type. Do this from the start.

Refine the helper input type:

```ts
type Deal = { logged_by: string | null };
```

Treat `null` defensively: `if (!deal.logged_by) return { label: "Unknown", variant: "unknown", ariaLabel: "Logged by unknown user" };`.
