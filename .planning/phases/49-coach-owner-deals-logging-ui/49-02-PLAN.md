# Plan 49-02 — Coach + Owner "Add Deal" wiring on student-detail Deals tab

## Objective

Add a single "Add Deal" primary button on both the coach and owner student-detail Deals tab that opens the **exact** same `DealFormModal` already used by students. Submitting the modal POSTs `{ revenue, profit, student_id }` to the existing `/api/deals` endpoint (Phase 45 — dual-layer authorization already in place). New row shows an attribution chip reading "You" for the logger. Optimistic insert + `router.refresh()` on both success and failure, matching the student DealsClient pattern verbatim.

## Scope (files)

**MODIFIED:**
- `src/components/coach/DealsTab.tsx` — add Add Deal button, modal state, POST handler, optimistic insert, accept `studentId`, `studentName`, `viewerId`, `viewerRole="coach"`, `userMap` props. Convert from pure presentational component to `"use client"`.
- `src/components/owner/OwnerStudentDetailClient.tsx` — pass `studentId`, `student.name`, `viewerId`, `viewerRole="owner"`, `userMap` to `DealsTab` (reuses the same coach `DealsTab` component per existing import).

- `src/app/(dashboard)/coach/students/[studentId]/page.tsx` — already fetches `userMap` + `viewerId` in plan 49-01; just pass them through.
- `src/app/(dashboard)/owner/students/[studentId]/page.tsx` — same.

### `handleAdd` implementation (mirror `DealsClient.handleAdd` verbatim, with student_id in body)

```ts
const handleAdd = useCallback(
  async (payload: { revenue: number; profit: number }) => {
    setSubmitting(true);

    const tempDeal: Deal = {
      id: String(-Date.now()),
      student_id: studentId,
      deal_number: optimisticDeals.length + 1,
      revenue: payload.revenue,
      profit: payload.profit,
      logged_by: viewerId,
      updated_by: null,
      created_at: new Date().toISOString(),
      updated_at: new Date().toISOString(),
    };

    startTransition(() => {
      dispatchOptimistic({ type: "add", deal: tempDeal });
    });

    setModalOpen(false);

    try {
      const res = await fetch("/api/deals", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ ...payload, student_id: studentId }),
      });

      if (!res.ok) {
        const err = await res.json().catch(() => ({ error: null }));
        toastRef.current({
          type: "error",
          title: (err as { error?: string }).error || "Failed to add deal",
        });
        routerRef.current.refresh();
      } else {
        toastRef.current({ type: "success", title: "Deal added" });
        routerRef.current.refresh();
      }
    } catch (err) {
      console.error("Failed to add deal:", err);
      toastRef.current({ type: "error", title: "Failed to add deal" });
      routerRef.current.refresh();
    } finally {
      setSubmitting(false);
    }
  },
  [optimisticDeals.length, dispatchOptimistic, studentId, viewerId],
);
```

Use stable refs `routerRef` / `toastRef` per CLAUDE.md useCallback dep pattern (same as `DealsClient`).

## API contract (already in place per Phase 45)

`POST /api/deals` with body `{ revenue: number, profit: number, student_id: string }`:
- Route handler (src/app/api/deals/route.ts) resolves `profile.role`.
- Coach branch: verifies `coach_id === profile.id` on target student → 403 if unassigned.
- Owner branch: verifies target is a student/student_diy → 404 if not.
- Admin client inserts: `{ student_id, revenue, profit, logged_by: profile.id }`.
- RLS WITH CHECK is the second layer (verified Phase 45).
- Revalidates `deals-${studentId}`, `student-analytics-${studentId}`, `coach-dashboard-${coachId}`, `coach-analytics-${coachId}` tags.

No API changes needed.

## Risk register

- **R1:** A coach tries to add a deal for a newly-unassigned student where the page was server-rendered before the unassignment. Expected outcome: POST returns 403, optimistic row disappears on `router.refresh()`, error toast shows. Acceptable — matches Phase 45 threat model T-45-03.
- **R2:** `studentName` contains quotes or special characters. Use template literal directly in `aria-label` — React escapes.
- **R3:** Double-click spam on Add Deal. `submitting` state disables the button through `Button loading={submitting}` prop propagated via modal; outer button can also be disabled when `submitting` is true — add `disabled={submitting}` to the outer Add Deal button.
