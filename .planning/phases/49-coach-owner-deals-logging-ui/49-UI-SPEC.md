## Interaction Contracts

### Add Deal (coach or owner view)
1. Click "Add Deal" button on student-detail Deals tab (44px target, `ima-primary`)
2. Shared `DealFormModal` opens (same as student) — `motion-safe:animate-scaleIn` on modal
3. Submit → `POST /api/deals` with `{ revenue, profit, student_id }` — server sets `logged_by` = session user
4. Optimistic row insert with attribution chip showing "You" for creator
5. `response.ok` check before JSON parse; `router.refresh()` on both success and failure
6. Toast: success "Deal added" / error "Failed to add deal"

### Attribution Chip Rendering (all 4 tables)
Gated ONLY by `formatDealLoggedBy(deal, viewerRole, viewerId, userMap)`:
- `deal.logged_by === viewerId` → chip label `"You"`, variant `self` (accent background)
- `logged_by` user role = `coach` and viewer is different → chip label `{firstName}`, variant `coach` (neutral)
- `logged_by` user role = `owner` and viewer is different → chip label `Owner: {firstName}`, variant `owner` (neutral)

Tables consuming the helper:
1. Student's own `DealsClient` deal list
2. Coach's student-detail `DealsTab` deal list
3. Owner's student-detail `DealsTab` deal list
4. Analytics history table (student + coach analytics pages)

### Animations (CLAUDE.md motion-safe rule)
- Modal open: `motion-safe:animate-scaleIn` + overlay `motion-safe:animate-fadeIn` (already in `tailwind.config.ts`)
- Optimistic row insert: no new animation — rely on React reconciliation; row appears instantly

## Accessibility Contract

- "Add Deal" button: `min-h-[44px]`, `aria-label="Add deal for {studentName}"` on coach/owner views; plain `Add Deal` label on student view (unchanged)
- Modal: `role="dialog"` + `aria-modal="true"` (inherited from shared `Modal` primitive)
- Attribution chip: `role="status"` + `aria-label="Logged by {fullName}"`; visible text is shorthand (`You` / `Michael` / `Owner: Abu Lahya`)
- Decorative icons (`Plus`, `DollarSign`): `aria-hidden="true"`
- Form inputs: existing `<label>` via `Input` primitive (already compliant)
- Error text: `role="alert"` on validation messages (already present in `DealFormModal`)
