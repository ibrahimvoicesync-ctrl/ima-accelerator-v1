    **Environment variables (passed via `--env` or `-e` flags):**
    - `APP_URL` — the staging Next.js app URL (e.g., `http://localhost:3000` or `https://staging.vercel.app`)

    **Token loading:**
    ```javascript
    import { SharedArray } from 'k6/data';
    import http from 'k6/http';
    import { check, sleep } from 'k6';

    const tokens = new SharedArray('student-tokens', function () {
      return JSON.parse(open('../tokens/student_tokens.json'));
    });
    ```

    **Options:**
    ```javascript
    export const options = {
      scenarios: {
        write_spike: {
          executor: 'ramping-vus',
          startVUs: 0,
          stages: [
            { duration: '2m', target: 500 },   // ramp to 500 VUs (simulates spike onset)
            { duration: '5m', target: 500 },   // sustain peak for 5 minutes
            { duration: '1m', target: 0 },     // ramp down
          ],
          gracefulRampDown: '30s',
        },
      },
      thresholds: {
        'http_req_duration': ['p(95)<1000'],   // D-11: P95 < 1 second
        'http_req_failed': ['rate<0.01'],       // error rate < 1%
      },
    };
    ```

    500 VUs (not 5,000) because each VU iterates multiple times over the 5-minute sustain period. With sleep(3), each VU makes ~100 requests over 5 min, so 500 VUs * ~100 = ~50,000 total requests — simulating 5,000 students submitting a few reports/sessions each.

    **Default function (per-VU iteration):**
    Each VU picks its unique token: `const token = tokens[(__VU - 1) % tokens.length];`

    Build headers object:
    ```javascript
    const headers = {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json',
      'Origin': __ENV.APP_URL,  // CSRF: must match app host (Pitfall 2)
    };
    ```

    Generate a random date string for the report (use today minus a random offset 0-89 to match seeded data range):
    ```javascript
    const dayOffset = Math.floor(Math.random() * 90);
    const d = new Date();
    d.setDate(d.getDate() - dayOffset);
    const date = d.toISOString().split('T')[0];
    ```

    **Request 1: Submit daily report (POST /api/reports)**
    Body must be Zod-valid:
    ```javascript
    {
      date: date,
      hours_worked: 2 + Math.random() * 6,         // 2.0-8.0
      star_rating: 1 + Math.floor(Math.random() * 5), // 1-5
      brands_contacted: Math.floor(Math.random() * 10),  // 0-9
      influencers_contacted: Math.floor(Math.random() * 5), // 0-4
      calls_joined: Math.floor(Math.random() * 3),  // 0-2
    }
    ```
    Check: status is 200 (update existing) or 201 (new) — both are success. Also accept 409 (duplicate) as non-failure.

    **Request 2: Start work session (POST /api/work-sessions)**
    Body must be Zod-valid:
    ```javascript
    {
      date: date,
      cycle_number: 1 + Math.floor(Math.random() * 10), // random cycle to avoid collisions
      session_minutes: [30, 45, 60][Math.floor(Math.random() * 3)],
    }
    ```
    Check: status is 201 (created) or 409 (active session exists / cycle duplicate) — both acceptable under load.

    **Add `sleep(3)` at the end of each iteration.** This keeps each VU under 30 req/min per endpoint (Pitfall 5: at 2 requests per iteration + 3s sleep, that is ~40 req/min total split across 2 endpoints = ~20 req/min per endpoint, safely under the 30/min limit).

       Same headers pattern.
       Body: `{}`
       Check: status 200

    3. **Paginated student list** — GET to `${__ENV.SUPABASE_URL}/rest/v1/users?role=eq.student&order=name.asc&offset=${randomPage * 25}&limit=25`
       Headers: same auth + apikey
       Check: status 200

    Random page offset: `Math.floor(Math.random() * 200)` (200 pages of 25 = 5000 students).

    Add `sleep(2)` between iterations (dashboard browsing cadence).
