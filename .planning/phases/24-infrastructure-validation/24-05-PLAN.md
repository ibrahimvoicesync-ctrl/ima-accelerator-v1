The measured values validate end-to-end correctness and relative performance, but are not directly comparable to production cloud infrastructure. The CAPACITY.md will note the environment difference.

Run the full local load test pipeline. Requires Docker Desktop running and a second terminal for Next.js.

**Option A: Fully automated (recommended)**

```bash

npm run dev

# Terminal 2: Run everything
bash load-tests/run-local.sh all
```

The script will:
1. Start local Supabase if not running (`npx supabase start`)
2. Check if DB is already seeded (skips if 5,000 students exist)
3. Seed the DB with 5,000 students + ~500k rows
4. Generate JWT tokens using the local JWT secret
5. Verify owner_token.json is a valid array
6. Prompt you to confirm Next.js is running
7. Run all 3 k6 scenarios with results saved to `load-tests/results/`
8. Capture post-test DB stats (connections, rate limits, row counts)

**Option B: Step by step**

```bash
bash load-tests/run-local.sh seed     # Seed only
bash load-tests/run-local.sh tokens   # Generate tokens only
bash load-tests/run-local.sh test     # Run k6 only (Next.js must be running)
```

**What to share back:**
