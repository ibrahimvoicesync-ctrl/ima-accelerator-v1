## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> POST /api/deals | Untrusted revenue/profit values from browser; student_id derived from auth, never from client |
| Client -> PATCH /api/deals/[id] | Untrusted id param and revenue/profit body; ownership enforced at query level |
| Client -> DELETE /api/deals/[id] | Untrusted id param; three-tier role check before delete |
| Client -> GET /api/deals | Untrusted student_id and page params; role restricted to coach/owner |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-39-01 | Spoofing | All 4 endpoints | mitigate | verifyOrigin(request) as first check in every handler; rejects missing or mismatched Origin header with 403 |
| T-39-02 | Tampering | POST revenue/profit | mitigate | Zod .min(0).max(9999999999.99) mirrors DB CHECK constraints; prevents negative values and numeric overflow |
| T-39-03 | Tampering | PATCH [id] param | mitigate | UUID regex validation before any DB call; malformed IDs return 400 |
| T-39-04 | Elevation of Privilege | PATCH /api/deals/[id] | mitigate | .eq("student_id", profile.id) in update query -- other students' deals silently excluded; returns 404 |
| T-39-05 | Elevation of Privilege | DELETE /api/deals/[id] | mitigate | Three-tier auth: student checks deal.student_id === profile.id, coach checks users.coach_id === profile.id, owner passthrough; all others 403 |
| T-39-06 | Information Disclosure | GET /api/deals | mitigate | Role check rejects student/student_diy with 403; only coach/owner can query by student_id |
| T-39-07 | Denial of Service | All 4 endpoints | mitigate | checkRateLimit(profile.id, endpoint) at 30 req/min default; 429 with Retry-After header |
| T-39-08 | Repudiation | All mutations | accept | No audit log in v1.5 scope; mutations are logged via revalidateTag and updated_at timestamps; acceptable at current scale |
| T-39-09 | Elevation of Privilege | GET student_id param | mitigate | student_id is a query param but GET is coach/owner only -- students cannot call GET to probe other students' deals |
| T-39-10 | Tampering | POST deal_number injection | mitigate | Insert payload omits deal_number entirely; BEFORE INSERT trigger always overwrites it regardless of what's sent |
</threat_model>
