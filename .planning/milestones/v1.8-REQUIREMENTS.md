## Out of Scope (explicit exclusions, reasoning preserved)

- Compat alias for old `total_emails` / `total_influencers` keys — breaking change intentional; keeping the aliases would silently preserve the double-count bug
- Parallel `/owner/students_diy/[studentId]` route tree — extended existing route via DIY-01 (one route tree, conditional rendering)
- Per-role sub-component `OwnerStudentDetailClientDIY.tsx` — inline conditional rendering inside existing component
- Tombstone message in alerts feed about the 4 legacy types — silent removal (tombstones age badly)
- Renaming `tech_setup` internal type key / dismissal-key prefix / config keys — only the UI label changed (internal rename would ripple through RPC, dismissal keys, in-flight dismissals)
- localStorage persistence of window selector — reset-on-visit is deliberate; URL-param is the v1.9+ upgrade path
- Coach route extension for DIY students — owner-only for v1.8
- Email / push / in-app notification extensions beyond the existing feeds — "notified" means "sees in feed"
- Changes to daily report form, student analytics trend charts, hours chart, or deals table
- Changes to `ROADMAP_STEPS` themselves — Step 4 "Set Up Your Agency" already exists
- `/owner/coaches/[id]` coach detail page — not added in v1.8
