CREATE OR REPLACE the `get_sidebar_badges` function. The entire function body must be reproduced (not just the new parts) because CREATE OR REPLACE replaces the whole function.

Header comment:
```
-- Phase 35: Chat badges — extend get_sidebar_badges with unread_messages
-- Migration: 00017_chat_badges.sql
-- Adds unread_messages count for coach and student roles
-- NOTE: Broadcast read_at tracks first reader only (v1 approximation, see RESEARCH.md open question #1)
```

The function signature stays the same: `get_sidebar_badges(p_user_id uuid, p_role text) RETURNS jsonb`.

Changes to the function body:
1. Add new variable: `v_unread_count integer := 0;` in DECLARE block
2. **Coach block** — after the existing `v_milestone_count - v_milestone_dismissed` calculation and BEFORE the RETURN, add:
   ```sql
   -- Unread messages: count of messages sent TO this coach (recipient_id = p_user_id) that are unread
   SELECT count(*) INTO v_unread_count
   FROM messages
   WHERE coach_id = p_user_id
     AND recipient_id = p_user_id
     AND read_at IS NULL;
   ```
   Then update the coach RETURN to include the new key:
   ```sql
   RETURN jsonb_build_object(
     'unreviewed_reports', v_unreviewed_count,
     'coach_milestone_alerts', GREATEST(0, v_milestone_count - v_milestone_dismissed),
     'unread_messages', v_unread_count
   );
   ```

3. **Student block** — add a NEW block after the coach block and before the owner block:
   ```sql
   IF p_role = 'student' THEN
     -- Unread DMs sent to this student + unread broadcasts from their coach
     SELECT count(*) INTO v_unread_count
     FROM messages m
     JOIN users u ON u.id = p_user_id
     WHERE m.coach_id = u.coach_id
       AND (
         (m.recipient_id = p_user_id AND m.read_at IS NULL)
         OR (m.is_broadcast = true AND m.read_at IS NULL)
       );
     RETURN jsonb_build_object('unread_messages', v_unread_count);
   END IF;
   ```
   Note: broadcast read_at is best-effort in v1 (single read_at column means first student to read marks it globally). This is documented and acceptable per research open question #1.

4. **Owner block** — remains EXACTLY as in 00014. Add `'unread_messages', 0` to the owner RETURN jsonb_build_object so the type contract is consistent (owner never has unread chat messages but the key should exist).

5. **Default/else** — at the very end (for student_diy or unknown roles): `RETURN '{}'::jsonb;`
