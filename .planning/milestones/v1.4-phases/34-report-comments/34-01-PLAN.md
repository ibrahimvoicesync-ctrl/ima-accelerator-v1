4. **Order of operations** (matches review route):
   - CSRF: `const csrfError = verifyOrigin(request); if (csrfError) return csrfError;`
   - Auth: `const supabase = await createClient(); const { data: { user: authUser } } = await supabase.auth.getUser(); if (!authUser) return 401;`
   - Admin client + profile lookup: `const admin = createAdminClient(); const { data: profile } = await admin.from("users").select("id, role").eq("auth_id", authUser.id).single(); if (!profile) return 404;`
   - **Role check (COMMENT-04, COMMENT-05):** `if (profile.role !== "coach" && profile.role !== "owner") return NextResponse.json({ error: "Forbidden" }, { status: 403 });`
   - Rate limit: `const { allowed, retryAfterSeconds } = await checkRateLimit(profile.id, "/api/reports/comment"); if (!allowed) return 429 with Retry-After header;`
   - Parse body with try/catch on `request.json()` — catch returns 400 "Invalid JSON body"
   - Zod safeParse — failure returns 400 with first issue message
   - Extract route param: `const { id } = await params;`
   - **Fetch report:** `const { data: report } = await admin.from("daily_reports").select("id, student_id").eq("id", id).single(); if (!report) return 404 "Report not found";`
   - **Ownership check (PITFALL 1 — conditional on role):** Only for coach role. Skip for owner.
     ```typescript
     if (profile.role === "coach") {
       const { data: studentMatch } = await admin
         .from("users").select("id").eq("id", report.student_id).eq("coach_id", profile.id).single();
       if (!studentMatch) return NextResponse.json({ error: "Report not found" }, { status: 404 });
     }
     ```
   - **Upsert (COMMENT-01, COMMENT-02):**
     ```typescript
     const { data, error } = await admin
       .from("report_comments")
       .upsert(
         { report_id: id, coach_id: profile.id, comment: parsed.data.comment },
         { onConflict: "report_id" }
       )
       .select()
       .single();
     if (error) return NextResponse.json({ error: error.message }, { status: 500 });
     return NextResponse.json({ data });
     ```

5. **Error handling:** Every catch block has `console.error` or returns an error response. No empty catch blocks. Every `fetch` (in this case, DB queries) checks for errors before proceeding.
