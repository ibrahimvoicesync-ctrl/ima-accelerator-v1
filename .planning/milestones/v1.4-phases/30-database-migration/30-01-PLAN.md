**Section 8: RLS policies -- report_comments**
Per D-04 (defense-in-depth), app layer handles primary access control. RLS is a safety net.

SELECT policies:
- `owner_select_report_comments`: owner can read all — `USING ((select get_user_role()) = 'owner')`
- `coach_select_report_comments`: coach can read comments on their assigned students' reports — `USING ((select get_user_role()) = 'coach' AND report_id IN (SELECT dr.id FROM public.daily_reports dr JOIN public.users s ON s.id = dr.student_id WHERE s.coach_id = (select get_user_id())))`
- `student_select_report_comments`: student can read comments on their own reports — `USING ((select get_user_role()) = 'student' AND report_id IN (SELECT dr.id FROM public.daily_reports dr WHERE dr.student_id = (select get_user_id())))`

INSERT policies:
- `owner_insert_report_comments`: owner can insert — `WITH CHECK ((select get_user_role()) = 'owner')`
- `coach_insert_report_comments`: coach can insert on their assigned students' reports — `WITH CHECK ((select get_user_role()) = 'coach' AND coach_id = (select get_user_id()) AND report_id IN (SELECT dr.id FROM public.daily_reports dr JOIN public.users s ON s.id = dr.student_id WHERE s.coach_id = (select get_user_id())))`

UPDATE policies:
- `owner_update_report_comments`: owner can update any — `USING ((select get_user_role()) = 'owner')`
- `coach_update_report_comments`: coach can update their own comments — `USING ((select get_user_role()) = 'coach' AND coach_id = (select get_user_id()))`

No DELETE policies (comments are upserted, not deleted).

**Section 9: RLS policies -- messages**
SELECT policies:
- `owner_select_messages`: owner can read all — `USING ((select get_user_role()) = 'owner')`
- `coach_select_messages`: coach reads messages where they are the coach_id — `USING ((select get_user_role()) = 'coach' AND coach_id = (select get_user_id()))`
- `student_select_messages`: student reads messages where they are recipient OR broadcasts from their coach — `USING ((select get_user_role()) = 'student' AND (recipient_id = (select get_user_id()) OR sender_id = (select get_user_id()) OR (is_broadcast = true AND coach_id = (SELECT coach_id FROM public.users WHERE id = (select get_user_id())))))`

INSERT policies:
- `coach_insert_messages`: coach inserts messages where coach_id = self and sender_id = self — `WITH CHECK ((select get_user_role()) = 'coach' AND coach_id = (select get_user_id()) AND sender_id = (select get_user_id()))`
- `student_insert_messages`: student inserts messages where sender_id = self and recipient_id IS NOT NULL and is_broadcast = false — `WITH CHECK ((select get_user_role()) = 'student' AND sender_id = (select get_user_id()) AND recipient_id IS NOT NULL AND is_broadcast = false)`

UPDATE policies (for read_at marking only):
- `coach_update_messages`: coach updates read_at on messages sent to them — `USING ((select get_user_role()) = 'coach' AND recipient_id = (select get_user_id()))`
- `student_update_messages`: student updates read_at on messages sent to them — `USING ((select get_user_role()) = 'student' AND (recipient_id = (select get_user_id()) OR (is_broadcast = true AND coach_id = (SELECT coach_id FROM public.users WHERE id = (select get_user_id())))))`

No DELETE policies (messages are never deleted in v1).
