**File 1: `src/app/(auth)/register/[code]/page.tsx`** -- Invite Code Registration

This is an async Server Component. It validates the invite code before rendering any UI.

**Server-side validation logic:**
```typescript
import { createAdminClient } from "@/lib/supabase/admin";

export default async function RegisterInvitePage({
  params,
  searchParams,
}: {
  params: Promise<{ code: string }>;
  searchParams: Promise<{ error?: string }>;
}) {
  const { code } = await params;
  const { error } = await searchParams;
```

Note: In Next.js 16, both `params` and `searchParams` are Promises and MUST be awaited.

**Validation steps (in the server component body):**
1. Regex check: `if (!/^[a-zA-Z0-9_-]{8,64}$/.test(code))` -- show invalid invite error card
2. Query invite: `admin.from("invites").select("id, email, role, code, used, expires_at").eq("code", code).single()`
3. If no invite found: show "This invite link is not valid" error card
4. If `invite.used`: show "This invite has already been used" error card
5. If `new Date(invite.expires_at) < new Date()`: show "This invite has expired. Ask your coach for a new one." error card
6. If valid: render RegisterCard client component with invite data

**Error card (reusable pattern for invalid states):**
- Same centered card layout as login page: `min-h-screen flex items-center justify-center bg-ima-bg`
- Card with `w-full max-w-md rounded-xl bg-ima-surface p-8 shadow-sm border border-ima-border text-center`
- Red error icon (lucide-react `XCircle`, 48px, `text-red-500`, `aria-hidden="true"`)
- Error heading and message
- "Back to Login" link: `min-h-[44px]` button-styled link to `/login`
