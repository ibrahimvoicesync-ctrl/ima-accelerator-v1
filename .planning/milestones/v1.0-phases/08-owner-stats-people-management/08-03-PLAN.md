Structure:

2. Admin client: `createAdminClient()`
3. Compute `sevenDaysAgo`:
   ```typescript
   const today = getToday();
   const nowMs = new Date(today + "T23:59:59Z").getTime();
   const sevenDaysAgo = new Date(nowMs - COACH_CONFIG.reportInboxDays * 24 * 60 * 60 * 1000).toISOString().split("T")[0];
   ```
4. Parallel fetch (3 queries via Promise.all):
   ```typescript
   const [coachesResult, studentsResult, reportsResult] = await Promise.all([
     admin.from("users").select("id, name, email, status, joined_at").eq("role", "coach").order("name"),
     admin.from("users").select("id, name, coach_id, status").eq("role", "student").eq("status", "active"),
     admin.from("daily_reports")
       .select("student_id, star_rating, date")
       .gte("date", sevenDaysAgo)
       .not("star_rating", "is", null),
   ]);
   ```
5. Error logging for each query result.
6. Build lookup maps in TypeScript:
   - `studentsByCoach: Map<string, number>` — iterate students, count per `coach_id`
   - `studentCoachMap: Map<string, string>` — map `student.id` -> `student.coach_id`
   - `coachRatings: Map<string, number[]>` — iterate reports, look up coach via studentCoachMap, collect ratings
7. Enrich coaches:
   ```typescript
   const enrichedCoaches = (coachesResult.data ?? []).map(coach => ({
     ...coach,
     studentCount: studentsByCoach.get(coach.id) ?? 0,
     avgRating: (() => {
       const ratings = coachRatings.get(coach.id);
       if (!ratings || ratings.length === 0) return null;
       return ratings.reduce((a, b) => a + b, 0) / ratings.length;
     })(),
   }));
   ```

   - Heading: `<h1 className="text-2xl font-bold text-ima-text">Coaches</h1>`
   - Subtitle: `<p className="mt-1 text-ima-text-secondary">{enrichedCoaches.length} coach{es} on the platform</p>`
   - Card grid: `grid grid-cols-1 md:grid-cols-2 gap-4 mt-6` (locked decision: 2-col desktop, 1-col mobile)
   - Each coach rendered via `<CoachCard key={coach.id} coach={coach} />`
   - Empty state when no coaches:

Create a single server component: `src/app/(dashboard)/owner/coaches/[coachId]/page.tsx`

This is a pure server component (no client component needed — no interactive elements). It follows the coach dashboard pattern: header + stat cards + student card grid.

Structure:

1. **Auth + params:**
   ```typescript
   import { requireRole } from "@/lib/session";
   import { createAdminClient } from "@/lib/supabase/admin";
   import { COACH_CONFIG } from "@/lib/config";
   import { getToday } from "@/lib/utils";
   import { notFound } from "next/navigation";
   import { Card, CardContent } from "@/components/ui/Card";
   import { Badge } from "@/components/ui/Badge";
   import { StudentCard } from "@/components/coach/StudentCard";
   import { Button } from "@/components/ui/Button";
   import { ChevronLeft, Users, Star, FileText, AlertTriangle } from "lucide-react";
   import Link from "next/link";

   export default async function CoachDetailPage({
     params,
   }: {
     params: Promise<{ coachId: string }>;
   }) {
     await requireRole("owner");
     const { coachId } = await params;
     const admin = createAdminClient();
   ```

2. **Fetch coach profile (single):**
   ```typescript
   const { data: coach, error: coachError } = await admin
     .from("users")
     .select("id, name, email, status, joined_at")
     .eq("id", coachId)
     .eq("role", "coach")
     .single();

   if (coachError) console.error("[owner coach detail] Failed to load coach:", coachError);
   if (!coach) notFound();
   ```

3. **Fetch assigned students:**
   ```typescript
   const { data: students, error: studentsError } = await admin
     .from("users")
     .select("id, name, email, status, joined_at")
     .eq("role", "student")
     .eq("coach_id", coachId)
     .eq("status", "active");

   if (studentsError) console.error("[owner coach detail] Failed to load students:", studentsError);
   const studentList = students ?? [];
   const studentIds = studentList.map(s => s.id);
   ```

4. **Parallel enrichment fetch** (same pattern as coach dashboard in Phase 6):
   If `studentIds.length > 0`, Promise.all for:
   - work_sessions: `select("student_id, date").in("student_id", studentIds).order("date", { ascending: false })`
   - daily_reports: `select("student_id, date, star_rating, reviewed_by, submitted_at").in("student_id", studentIds)`
   - roadmap_progress: `select("student_id, step_number, status").in("student_id", studentIds)`
   Else use empty results `{ data: null, error: null }`.

5. **Build lookup maps** (identical to coach dashboard pattern):
   - `latestSessionMap: Map<string, string>` — latest session date per student
   - `latestReportMap: Map<string, string>` — latest report date per student
   - `todayReportMap: Map<string, boolean>` — students who submitted today
   - `recentRatings: Map<string, number[]>` — star ratings in last 7 days
   - `roadmapStepMap: Map<string, number>` — highest completed/active step per student

6. **Compute 4 coach-level stat cards:**
   - **Student Count:** `studentList.length`
   - **Avg Student Rating (7-day):** Flatten all recentRatings values, compute average. `allRatings.length > 0 ? (sum / allRatings.length).toFixed(1) : "—"`
   - **Report Review Rate (%):** From reports in last 7 days:
     ```typescript
     const sevenDaysAgo = new Date(nowMs - COACH_CONFIG.reportInboxDays * 24 * 60 * 60 * 1000).toISOString().split("T")[0];
     let submittedCount = 0;
     let reviewedCount = 0;
     for (const r of reportsData) {
       if (r.date >= sevenDaysAgo && r.submitted_at !== null) {
         submittedCount++;
         if (r.reviewed_by !== null) reviewedCount++;
       }
     }
     const reviewRate = submittedCount > 0 ? Math.round((reviewedCount / submittedCount) * 100) : 0;
     ```
   - **At-Risk Count:** Count students from enriched list where `isAtRisk === true`

7. **Enrich students for StudentCard** (identical logic to coach dashboard's enriched student computation):
   - For each student: compute isNew, isAtRisk, atRiskReasons, lastActiveLabel, todayReportSubmitted, currentRoadmapStep
   - Sort: at-risk first, then alphabetical
   - Derive `nowMs` from `getToday()` string: `const nowMs = new Date(today + "T23:59:59Z").getTime();`

8. **Render:**

   a. **Header** (locked decision: back link, initials avatar, coach name, email):
   ```tsx
   <div className="px-4">
     <Link href="/owner/coaches">
       <Button variant="ghost" size="sm">
         <ChevronLeft className="h-4 w-4 mr-1" aria-hidden="true" />
         Back to Coaches
       </Button>
     </Link>

     <div className="flex items-center gap-4 mt-4">
       <div className="w-14 h-14 rounded-full bg-ima-primary flex items-center justify-center text-xl font-bold text-white shrink-0">
         {coach.name.charAt(0).toUpperCase()}
       </div>
       <div>
         <h1 className="text-xl font-bold text-ima-text">{coach.name}</h1>
         <p className="text-sm text-ima-text-secondary">{coach.email}</p>
       </div>
     </div>
   ```

   b. **4 Stat Cards** in `grid grid-cols-2 lg:grid-cols-4 gap-4 mt-6`:
   - Student Count: `Users` icon, `text-ima-primary`, value = `studentList.length`
   - Avg Rating: `Star` icon, `text-ima-primary`, value = avgRating string
   - Review Rate: `FileText` icon, `text-ima-primary`, value = `${reviewRate}%`
   - At-Risk: `AlertTriangle` icon, `text-ima-warning`, value = atRiskCount (text color: `atRiskCount > 0 ? "text-ima-error" : "text-ima-text"`)

   ```tsx
         <p className="text-xs text-ima-text-secondary">{label}</p>
       </div>
     </CardContent>
   </Card>
   ```
   For the At-Risk card, use `bg-ima-warning/10` and `text-ima-warning` for the icon instead.

   c. **Assigned Students section:**
