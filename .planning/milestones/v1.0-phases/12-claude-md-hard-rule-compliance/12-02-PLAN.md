**Fix 2: getToday() UTC bug in src/lib/utils.ts (lines 9-11)**

Current code:
```typescript
/** Returns today's date as YYYY-MM-DD in local time */
export function getToday(): string {
  return new Date().toISOString().split("T")[0];
}
```

Replace with:
```typescript
/** Returns today's date as YYYY-MM-DD in local time */
export function getToday(): string {
  const d = new Date();
  const year = d.getFullYear();
  const month = String(d.getMonth() + 1).padStart(2, "0");
  const day = String(d.getDate()).padStart(2, "0");
  return `${year}-${month}-${day}`;
}
```

Why: `toISOString()` returns UTC. For a UAE-based cohort (UTC+4), sessions started between 00:00 and 03:59 local time would get yesterday's date. The fix uses `getFullYear()/getMonth()/getDate()` which return local-time values. Since `getToday()` is only called from client components (WorkTrackerClient.tsx), the browser provides the user's local timezone.

Do NOT change any other function in utils.ts. The JSDoc comment stays the same since it already says "local time" (the implementation now matches the documentation).
