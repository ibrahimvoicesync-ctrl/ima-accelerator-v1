### PARTIAL / MAJOR (1)

#### [MAJOR] work-sessions PATCH missing coach-cache bust
- **File:** `src/app/api/work-sessions/[id]/route.ts`
- **Issue:** PATCH to `status='completed'` fires `revalidateTag("badges")` + `revalidateTag(studentAnalyticsTag)` only. Does NOT fire `coachDashboardTag` or `coachAnalyticsTag`.
- **Why it matters:** `get_coach_dashboard` and `get_coach_analytics` both query `work_sessions WHERE status='completed'`. The session completion PATCH is the primary data-change event for the "Top 3 hours this week" leaderboard.
- **Impact:** Coach dashboard/analytics leaderboards stale up to 60s after a student completes a session (self-heals on TTL expiry).
- **Affected REQ-IDs:** COACH-DASH-04, COACH-DASH-06, COACH-ANALYTICS-07, PERF-05
- **Severity:** major (functional impact but self-healing — not a data-correctness bug)
- **Note:** `work-sessions/route.ts` POST (session creation, status='in_progress') DOES fire both coach tags, but at that point the session hasn't completed and wouldn't appear in the leaderboard. The invalidation is on the wrong verb.

### MINOR (2)

#### [MINOR] CSV export missing rate limit
- **File:** `src/app/api/coach/analytics/export.csv/route.ts`
- **Issue:** Route has auth + role check, no `checkRateLimit`. GET with no state mutation so CSRF N/A.
- **Affected REQ-IDs:** PERF-02
- **Impact:** Coach could loop 5000-row RPC in tight loop.

#### [MINOR] Orphaned deals-${studentId} revalidateTag
- **File:** `src/app/api/deals/route.ts` (lines 183, 213)
- **Issue:** Tag is fired but `/student/deals` and `/student_diy/deals` are direct-fetch server components with no `unstable_cache` + `tags:` registration. Dead invalidation.
- **Affected REQ-IDs:** DEALS-09 (functionality works via direct fetch — not a break)
- **Impact:** None — just dead work.
