## v1.6 Requirements — All Satisfied

> Note: Some checkboxes in the original REQUIREMENTS.md remained unchecked at milestone close. The milestone audit cross-referenced all 35 REQ-IDs against each phase's VERIFICATION.md (the authoritative source) and confirmed every requirement PASSED. This archive reflects verified status.

### Owner Analytics (OA)

- [x] **OA-01** — Owner can view `/owner/analytics` page (separate route from owner dashboard homepage)
- [x] **OA-02** — Owner Analytics page shows 3 top-3 leaderboards: Top 3 Students by Hours Worked, Top 3 Students by Profit Earned, Top 3 Students by Deals Closed (lifetime scope, deterministic tie-break)
- [x] **OA-03** — Owner dashboard homepage shows teaser stat cards linking to `/owner/analytics`
- [x] **OA-04** — Owner Analytics served by a single batch Postgres RPC (`get_owner_analytics`) wrapped in `unstable_cache` with 60s TTL
- [x] **OA-05** — `owner-analytics` cache tag is invalidated on every `deals` mutation (INSERT/UPDATE/DELETE) and on work-session completion (PATCH → completed)
- [x] **OA-06** — Leaderboard rows link to `/owner/students/[studentId]`

### Chat Removal (CHAT-REM)

- [x] **CHAT-REM-01** — All chat routes (page + API) removed from `src/app/`
- [x] **CHAT-REM-02** — All chat UI components removed from `src/components/chat/` and related files
- [x] **CHAT-REM-03** — All chat entries removed from sidebar nav in `src/lib/config.ts` (all roles)
- [x] **CHAT-REM-04** — All chat-related types, config entries, and utility files removed (`src/lib/chat-utils.ts` etc.)
- [x] **CHAT-REM-05** — `messages` table dropped in migration (clean removal, no archive)
- [x] **CHAT-REM-06** — `get_sidebar_badges` RPC rewritten to remove `unread_messages` branch — same migration and transaction as `DROP TABLE messages`
- [x] **CHAT-REM-07** — `SidebarBadgesResult` TypeScript type no longer contains `unread_messages`; all call-sites updated
- [x] **CHAT-REM-08** — `src/proxy.ts` route entries for `/coach/chat` and `/student/chat` removed

### Announcements (ANNOUNCE)

- [x] **ANNOUNCE-01** — New `announcements` table: `id`, `author_id` FK→`users`, `content` (text), `created_at`, `updated_at`
- [x] **ANNOUNCE-02** — Owner and coach can create announcements
- [x] **ANNOUNCE-03** — Owner and coach can edit ANY announcement (not just their own)
- [x] **ANNOUNCE-04** — Owner and coach can delete ANY announcement (not just their own)
- [x] **ANNOUNCE-05** — All students (role `student`) see all announcements — read-only, not scoped to their assigned coach
- [x] **ANNOUNCE-06** — `student_diy` role sees all announcements — read-only (same feed as students)
- [x] **ANNOUNCE-07** — Each role has a dedicated `/announcements` page accessible from sidebar nav (owner, coach, student, student_diy)
- [x] **ANNOUNCE-08** — Announcements list is paginated at 25 per page, most recent first
- [x] **ANNOUNCE-09** — Announcements never expire (persist forever)
- [x] **ANNOUNCE-10** — RLS policies: students + student_diy SELECT only; owner + coach have full CRUD on all rows; policies use `(SELECT auth.uid())` initplan pattern
- [x] **ANNOUNCE-11** — Announcement mutation routes (POST / PATCH / DELETE) are auth-gated, role-checked, and rate-limited at 30 req/min per user
- [x] **ANNOUNCE-12** — Edited announcements show an "(edited)" indicator based on `updated_at > created_at`

### Roadmap Step 8 Insertion (ROADMAP)

- [x] **ROADMAP-01** — New step inserted at end of Stage 1: title "Join at least one Influencer Q&A session (CPM + pricing)", `target_days: 5`
- [x] **ROADMAP-02** — Existing Steps 8–15 renumbered to 9–16 atomically in a single transaction with no duplicate `step_number` visible mid-transaction
- [x] **ROADMAP-03** — Students who completed old Step 7 have new Step 8 auto-marked `completed`
- [x] **ROADMAP-04** — Students can self-mark new Step 8 complete with the same flow as all other roadmap steps
- [x] **ROADMAP-05** — `ROADMAP_STEPS` in `src/lib/config.ts` updated to 16 entries with correct stage headers
- [x] **ROADMAP-06** — `MILESTONE_CONFIG.influencersClosedStep` updated 11→12; `brandResponseStep` 13→14 — in both `src/lib/config.ts` AND `get_coach_milestones` RPC SQL, in the same migration
- [x] **ROADMAP-07** — All progress-bar denominators show /16 (via dynamic `ROADMAP_STEPS.length`)
- [x] **ROADMAP-08** — Full grep sweep across codebase for hardcoded step numbers; zero offending literals in `src/`
- [x] **ROADMAP-09** — `CHECK (step_number BETWEEN 1 AND 15)` constraint dropped and recreated as `BETWEEN 1 AND 16` in the migration

### Performance & Quality (PERF)

- [x] **PERF-03** — All new RLS policies use `(SELECT auth.uid())` initplan pattern
